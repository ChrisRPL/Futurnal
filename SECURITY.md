# Security Policy

## Supported Versions

Security fixes are applied to the default branch.

## Reporting a Vulnerability

Please report vulnerabilities privately.

- Email: `steipete@gmail.com`
- Subject: `Futurnal Security Report`

Include:

- Impact summary
- Reproduction steps
- Affected files/components
- Suggested mitigation (if available)

Please do not open public GitHub issues for security vulnerabilities.

## Disclosure Process

1. We acknowledge receipt.
2. We triage and reproduce.
3. We prepare and validate a fix.
4. We publish a patch and coordinated disclosure notes.
